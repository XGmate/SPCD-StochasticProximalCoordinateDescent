# SPCD-StochasticProximalCoordinateDescent
Detailed information and additional extended experiments regarding the SPCD: Stochastic Proximal Coordinate Descent for Efficient and Robot Safe Path Smoothing submitted to RAL.

# Supplement 1
To further illustrate the impact of the batch size on algorithm performance, we present here additional experiments with batch ratios of 0.1, 0.3, 0.5, 0.7, and 1.0. (The paper provides comparison results for batch ratios of 0.5 and 0.6 due to space limitations.) The convergence curves and runtime data are available under 
**BatchRatio_Effect_On_Convergence** in the link.

Key information:

(I) A larger batch ratio accelerates convergence (e.g., a batch ratio of 0.7 reaches a stable cost after ~80 iterations, while a batch ratio of 0.3 requires 150 iterations).

(II) Runtime increases nearly linearly with the batch ratio (avg. time: batch ratio 0.1 → 0.70 s; batch ratio 0.5 → 1.23 s; batch ratio 1.0 → 1.88 s per run).

(III) Batch ratios of 0.5 or 0.6 offer the best trade-off: they achieve fast convergence with moderate runtime, while maintaining a safety margin comparable to larger batch ratios.

# Supplement 2
To evaluate its behavior in extreme non‑convex scenarios, we conducted additional experiments in a U‑shaped and a maze‑like environment (see **SPCD_Nonconvex_Validation**). SPCD successfully kept the path outside the U‑shaped trap, achieving a minimum safety distance 5.0 times larger than the best baseline. In the maze‑like environment, it maintained the original topology and produced a minimum safety distance 3.5 times larger than the best baseline while preserving path length and curvature. These results confirm that SPCD does not alter path topology and is robust against local minima even in highly non‑convex settings. The adaptive step size and stochastic coordinate updates further help escape shallow local minima. 

# Supplement 3
The SPCD method is not limited to single-agent scenarios. It can be extended to multi-agent scenarios. SPCD, like B‑spline or cubic spline, is inherently agnostic to the number of agents. Multi‑agent coordination is handled by higher‑level planning that provides collision‑free reference paths; SPCD then refines each path locally while preserving topology. To demonstrate, we conducted experiments with 3, 5, and 7 robots in a shared environment, each path generated by a centralized planner and SPCD applied individually. Results (see **SPCD_Multi-Robot_Scenario_Verification**) show SPCD successfully smooths all paths while maintaining path consistency.
